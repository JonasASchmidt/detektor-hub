12. For segments that look like UUIDs (regex: `/^[0-9a-f]{8}-/` or any segment not in SEGMENT_LABELS that appears after "findings"), resolve to the finding name via a lightweight client-side fetch.
13. Implement with a `useFindingName(id: string | null)` hook inside the same file or a small helper:
    ```typescript
    function useFindingName(id: string | null) {
      const [name, setName] = useState<string | null>(null);
      useEffect(() => {
        if (!id) return;
        fetch(`/api/findings/${id}`)
          .then(res => res.ok ? res.json() : null)
          .then(data => { if (data?.name) setName(data.name); })
          .catch(() => {});
      }, [id]);
      return name;
    }
    ```
14. When rendering segments, check if the segment is a dynamic ID (not in SEGMENT_LABELS). If so, use the resolved name from the hook. While loading, show a short placeholder (e.g., the first 8 chars of the UUID with `...`). Once resolved, show the finding name.

<Collapsible open={openState[item.title]} onOpenChange={(open) => setOpenState(prev => ({ ...prev, [item.title]: open }))}>
