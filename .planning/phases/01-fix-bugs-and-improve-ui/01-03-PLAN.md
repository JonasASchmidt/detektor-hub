**Extend `app/api/findings/route.ts` GET handler:**
Add support for these new query params:
- `status` (string: "DRAFT" or "COMPLETED") -- filter by FindingStatus enum
- `dateFrom` (ISO date string) -- filter `foundAt >= dateFrom`
- `dateTo` (ISO date string) -- filter `foundAt <= dateTo`
- `tags` (comma-separated tag IDs) -- filter findings that have ANY of the specified tags (support multiple, not just single `tag` param)
Keep backward compatibility with the existing `tag` single param.

Update the Prisma `where` clause to include these filters:
```typescript
const status = searchParams.get("status");
const dateFrom = searchParams.get("dateFrom");
const dateTo = searchParams.get("dateTo");
const tagsParam = searchParams.get("tags"); // comma-separated
const tagIds = tagsParam ? tagsParam.split(",") : (tag ? [tag] : []);

// Add to AND array:
...(status ? [{ status: status as FindingStatus }] : []),
...(dateFrom ? [{ foundAt: { gte: new Date(dateFrom) } }] : []),
...(dateTo ? [{ foundAt: { lte: new Date(dateTo) } }] : []),
...(tagIds.length > 0 ? [{ tags: { some: { id: { in: tagIds } } } }] : []),
```
Also fix the `total` count to use the same `where` clause (currently it counts ALL findings regardless of filters).
