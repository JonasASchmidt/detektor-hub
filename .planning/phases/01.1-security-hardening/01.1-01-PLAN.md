<objective>
Add authentication guards to all tag and category mutation routes (POST/PUT on tags, POST on tag-categories, DELETE on tag-categories/[id]) and fix the inconsistent getServerSession usage in the bulk images route.

Purpose: Tag and category creation/modification is currently unauthenticated — any anonymous request can create or delete tags. This closes that security gap.
Output: All tag/category mutation routes return 401 for unauthenticated requests. Bulk images route uses getServerSession(authOptions) consistently.
</objective>

<tasks>

<task type="code">
  <name>Task 1: Add auth to /api/tags POST and PUT</name>
  <action>
Add `import { getServerSession } from "next-auth"` and `import { authOptions } from "@/lib/auth"` to `app/api/tags/route.ts`.

At the start of the POST handler body (before reading req.json()), add:
```ts
const session = await getServerSession(authOptions);
if (!session?.user?.id) {
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}
```

Add the identical block at the start of the PUT handler body.

<task type="code">
  <name>Task 3: Fix getServerSession in /api/images/bulk</name>
  <action>
In `app/api/images/bulk/route.ts`:
1. Change `import { getServerSession } from "next-auth/next"` to `import { getServerSession } from "next-auth"`
2. Add `import { authOptions } from "@/lib/auth"`
3. Change `import { prisma } from "@/lib/prisma"` to `import prisma from "@/lib/prisma"` (default export)
4. Replace the auth block:
```ts
// BEFORE:
const session = await getServerSession();
if (!session?.user?.email) { ... }
const user = await prisma.user.findUnique({ where: { email: session.user.email } });
if (!user) { ... }
```
With:
```ts
// AFTER:
const session = await getServerSession(authOptions);
if (!session?.user?.id) {
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}
const userId = session.user.id;
```
5. Replace all remaining `user.id` references in the handler body with `userId`.
  </action>
